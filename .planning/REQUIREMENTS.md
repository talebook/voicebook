# book2audio 需求文档

## 1. 需求概览

| 项目 | 描述 |
|------|------|
| **项目名称** | book2audio 智能有声书工具 |
| **版本** | v1.0 |
| **核心功能** | 将小说TXT转化为多角色有声书，动态匹配音色 |

---

## 2. 功能需求

### 2.1 v1.0 需求（Active）

#### 核心功能

| ID | 需求 | 优先级 | 描述 |
|----|------|--------|------|
| **REQ-001** | TXT文本解析 | P0 | 解析小说TXT文件，自动章节分割 |
| **REQ-002** | 角色识别 | P0 | 使用LLM识别文本中的角色 |
| **REQ-003** | 对话/旁白分类 | P0 | 自动区分对话、旁白、描述性文字 |
| **REQ-004** | 角色画像提取 | P0 | 提取角色年龄、外貌、气质、音色描述 |
| **REQ-005** | TTS语音合成 | P0 | 支持多音色TTS，生成语音 |
| **REQ-006** | 音频拼接 | P0 | 将多个音频片段拼接成完整章节 |
| **REQ-007** | 年龄-音色映射 | P1 | 角色年龄变化时自动切换音色 |

#### 数据处理

| ID | 需求 | 优先级 | 描述 |
|----|------|--------|------|
| **REQ-008** | 角色历史追踪 | P0 | 跨章节追踪角色状态变化 |
| **REQ-009** | 发言归属 | P1 | 将对话准确归属到对应角色 |

#### 输出功能

| ID | 需求 | 优先级 | 描述 |
|----|------|--------|------|
| **REQ-010** | 多格式输出 | P1 | 支持MP3/M4A/WAV格式 |
| **REQ-011** | 分章节输出 | P1 | 可按章节独立生成音频 |

### 2.2 v2.0 需求（Deferred）

| ID | 需求 | 优先级 | 描述 |
|----|------|--------|------|
| **REQ-012** | 声音克隆 | P2 | 基于参考音频克隆特定音色 |
| **REQ-013** | 情感TTS | P2 | 根据情绪状态调整语调和语速 |
| **REQ-014** | 背景音乐 | P2 | 自动添加场景氛围背景音乐 |
| **REQ-015** | GUI界面 | P3 | 图形化操作界面 |

---

## 3. 非功能需求

### 3.1 性能约束

| 指标 | 要求 | 说明 |
|------|------|------|
| **LLM内存** | ≤1GB | Qwen3-0.6B或其他1B以下模型 |
| **模型大小** | ≤1B参数 | 推荐使用量化模型 |
| **单章节分析** | <30秒 | LLM角色分析耗时 |
| **单章节TTS** | <5分钟 | 语音合成耗时 |

### 3.2 技术约束

| 约束 | 要求 |
|------|------|
| **开发语言** | Python 3.10+ |
| **LLM框架** | Ollama / Transformers |
| **TTS方案** | 本地部署（Coqui XTTS / CosyVoice / Edge TTS） |
| **音频格式** | UTF-8 TXT输入，MP3/M4A输出 |

---

## 4. 技术选型需求

### 4.1 LLM模型

| 需求 | 说明 |
|------|------|
| **参数量** | ≤1B（内存占用≤1GB） |
| **中文支持** | 必须 |
| **推荐模型** | Qwen3-0.6B（已有）、Qwen2.5-0.5B |

### 4.2 TTS方案

| 方案 | 音色 | 中文 | 部署 | 优先级 |
|------|------|------|------|--------|
| **Coqui XTTS v2** | 多音色+克隆 | 需微调 | 本地 | P0 |
| **CosyVoice** | 多音色 | 原生支持 | 本地 | P0 |
| **Edge TTS** | 10+音色 | 支持 | 云API | P1 |
| **Azure TTS** | 100+音色 | 支持 | 云API | P2 |

## 5. 数据模型需求

### 5.1 角色画像

```python
@dataclass
class CharacterProfile:
    name: str              # 角色名称
    chapter: int            # 章节号
    age_stage: str         # 年龄阶段: 童年/少年/青年/中年/老年
    age_years: int        # 具体年龄（推测）
    gender: str            # 性别
    appearance: str        # 外貌特征
    temperament: str        # 气质性格
    voice_type: str        # 音色类型
    emotional_state: str   # 当前情绪
    key_changes: str       # 与上一章节的变化
```

### 5.2 对话记录

```python
@dataclass
class DialogueEntry:
    speaker: str           # 说话者
    content: str           # 对话内容
    chapter: int          # 章节号
    paragraph_idx: int    # 段落索引
    emotion: str           # 情绪（可选）
```

## 6. API需求

### 6.1 核心接口

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/analyze` | POST | 分析章节角色 |
| `/api/convert` | POST | 转换TXT到音频 |
| `/api/character/{name}/arc` | GET | 获取角色发展轨迹 |
| `/api/voices` | GET | 获取可用音色列表 |
| `/api/health` | GET | 健康检查 |

### 6.2 请求/响应格式

```json
// POST /api/convert
{
    "file_path": "novel.txt",
    "chapters": [1, 2, 3],  // 可选，默认全部
    "voice_mapping": {
        "张三": "female_youth",
        "李四": "male_adult"
    }
}

// 响应
{
    "status": "success",
    "output_files": ["chapter_1.mp3", "chapter_2.mp3"],
    "characters": ["张三", "李四"]
}
```

## 7. 用户流程

### 7.1 完整流程

```
1. 用户上传小说TXT
         ↓
2. 系统解析章节
         ↓
3. LLM识别角色 + 对话
         ↓
4. 用户确认/调整角色音色映射
         ↓
5. TTS批量合成
         ↓
6. 音频拼接 + 输出
```

### 7.2 快速流程

```
1. 用户上传小说TXT
         ↓
2. 系统全自动处理
         ↓
3. 输出有声书
```

## 8. 验收标准

### 8.1 功能验收

| ID | 标准 | 测试方法 |
|----|------|---------|
| **V-001** | 能解析100章以内的小说TXT | 输入测试文件，验证章节数 |
| **V-002** | LLM识别角色准确率≥80% | 人工对比10章节 |
| **V-003** | 对话/旁白分类准确率≥85% | 随机抽样验证 |
| **V-004** | TTS生成可理解语音 | 人工试听 |
| **V-005** | 音频拼接无明显断层 | 人工试听全章节 |

### 8.2 性能验收

| ID | 标准 | 测试方法 |
|----|------|---------|
| **V-006** | LLM内存占用≤1GB | 监控进程内存 |
| **V-007** | 单章节总耗时<6分钟 | 计时验证 |

## 9. Out of Scope

| 功能 | 排除原因 |
|------|---------|
| **视频生成** | 超出当前项目范围 |
| **实时语音流** | 需要WebSocket，非核心需求 |
| **多语言支持** | 聚焦中文有声书 |
| **版权检测** | 用户需自行确保版权合规 |
| **复杂音效** | v2考虑，先做基础TTS |
