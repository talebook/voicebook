# book2audio 智能有声书工具

## 1. 核心价值

**一句话描述**: 将小说TXT文件转化为多角色有声书，根据角色年龄和性格动态匹配音色，实现年龄变化时的音色自然过渡。

**解决的问题**:
- 有声书制作成本高昂（专业配音演员）
- 现有TTS缺乏角色个性，同质化严重
- 角色年龄跨度大时（如主角从少年到老年）音色无法自然过渡

---

## 2. 项目概述

### 2.1 目标

开发一个端到端的智能有声书工具，能够：
1. **输入**: 小说TXT文件
2. **分析**: 使用LLM识别角色、对话、旁白，提取角色画像
3. **合成**: 根据角色画像动态调整TTS参数，生成个性化音频
4. **输出**: 多角色有声书音频文件

### 2.2 约束条件

| 约束项 | 要求 |
|--------|------|
| **LLM内存占用** | ≤1GB |
| **模型大小** | 推荐 ≤1B 参数 |
| **TTS质量** | 支持中文，多音色可选 |
| **运行环境** | 本地/服务器均可 |
| **开发语言** | Python |

### 2.3 核心特性

- [ ] 小说文本解析（章节划分、段落分类）
- [ ] 角色识别与画像提取
- [ ] 对话/旁白自动识别
- [ ] 角色年龄追踪与音色切换
- [ ] TTS多音色支持
- [ ] 音频拼接与输出

## 3. 用户场景

### 3.1 主要用户

- 有声书制作团队
- 个人创作者/自媒体

### 3.2 典型工作流

```
小说TXT → 文本预处理 → LLM角色分析 → 对话提取 → TTS语音合成 → 音频拼接 → 有声书输出
   │           │            │              │           │            │
   ▼           ▼            ▼              ▼           ▼            ▼
 原始输入    章节分段落   角色画像       对话+旁白    多角色音频     MP3/M4A
```

## 4. 关键技术决策

### 4.1 LLM选型（内存≤1GB）

| 模型 | 参数量 | 内存占用 | 推荐度 |
|------|--------|---------|--------|
| **Qwen3-0.6B** | 0.6B | ~600MB | ⭐⭐⭐⭐⭐ (已有) |
| Qwen2.5-0.5B | 0.5B | ~500MB | ⭐⭐⭐⭐ |
| Phi-3-mini | 3.8B | ~4GB | ⭐⭐ (超出限制) |
| Llama-3.2-1B | 1B | ~1GB | ⭐⭐⭐ |

**决策**: 使用 **Qwen3-0.6B** 作为核心LLM（已有项目基础）

### 4.2 TTS选型

| TTS方案 | 音色数量 | 中文支持 | 部署方式 | 推荐度 |
|---------|---------|---------|----------|--------|
| **Coqui XTTS** | 多音色 | 需微调 | 本地 | ⭐⭐⭐⭐ |
| **VALL-E X** | 声音克隆 | 支持 | 本地 | ⭐⭐⭐ |
| **CosyVoice** | 多音色 | 原生 | 本地 | ⭐⭐⭐⭐⭐ |
| **Azure TTS** | 100+ | 支持 | 云API | ⭐⭐⭐ |
| **Edge TTS** | 10+ | 支持 | 免费API | ⭐⭐⭐⭐ |

**决策**: 优先 **Coqui XTTS v2** 或 **CosyVoice**（本地部署，支持音色克隆）

### 4.3 技术栈

| 层级 | 技术选型 |
|------|---------|
| **LLM推理** | Ollama + Qwen3-0.6B |
| **TTS合成** | Coqui XTTS v2 / CosyVoice |
| **音频处理** | pydub, soundfile |
| **后端框架** | FastAPI (扩展现有服务) |
| **数据处理** | Python 标准库 + 正则 |

## 8. 风险与假设

### 8.1 已知风险

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| Qwen3-0.6B 理解能力有限 | 角色识别准确率 | Prompt工程优化 + 后处理 |
| 中文TTS质量 | 听感体验 | 测试多个TTS方案 |
| 长文本处理 | 上下文丢失 | 分段处理 + 角色状态传递 |

### 8.2 假设

- 运行环境: 有GPU更好（加速TTS），无GPU也可运行
- 输入文本: UTF-8编码的TXT文件
- 网络: TTS如用云服务需联网，本地模型无需网络
