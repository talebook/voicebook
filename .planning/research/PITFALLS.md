# book2audio 常见陷阱

## 1. LLM 相关陷阱

### 1.1 角色识别准确率不足

**问题**: Qwen3-0.6B 参数量小，对复杂角色关系理解不足

**表现**:
- 遗漏次要角色
- 错误识别同名角色
- 角色关系混乱

**预防策略**:
1. Prompt 工程优化
   - 明确要求逐段落分析
   - 要求输出置信度
   - 提供角色列表参考
2. 后处理校验
   - 验证角色名一致性
   - 过滤异常识别结果
3. 上下文增强
   - 传递上一章角色列表
   - 维护角色关系图谱

**检测方法**:
```python
# 检查识别置信度
if profile.confidence < 0.6:
    flag_for_review(profile)
```

---

### 1.2 长文本上下文丢失

**问题**: 单次调用无法处理整本小说

**表现**:
- 章节开头角色状态不一致
- 重复询问角色背景

**预防策略**:
1. 分段处理策略
   - 按章节处理
   - 传递角色历史状态
2. 状态传递机制
   - 维护 CharacterHistory
   - Prompt 中包含历史摘要

---

## 2. 中文对话识别陷阱

### 2.1 对话标记识别

**问题**: 中文小说格式不统一

**常见格式**:
```
"张三说："你好。""
张三大声道："快跑！"
「李四」：听说...
张三: "你好"
```

**预防策略**:
1. 多规则并行匹配
2. LLM 二次验证
3. 用户可编辑映射表

---

### 2.2 说话者归属错误

**问题**: 连续对话无明确归属

**示例**:
```
"谢谢！"张三笑着接过。
"不客气。"李四回答。
"我们去哪？"张三问。
```

**预防策略**:
1. 追踪最近说话者
2. 考虑对话顺序和语义
3. 标记不确定片段供用户确认

---

## 3. TTS 相关陷阱

### 3.1 中文发音错误

**问题**: 专有名词、人名读音错误

**预防策略**:
1. 维护发音词典
2. LLM 生成拼音标注
3. 后处理修正

```python
# 发音词典
PRONUNCIATION_MAP = {
    "甄嬛": "zhēn huán",
    "阿房宫": "ē páng gōng"
}
```

---

### 3.2 音色不连贯

**问题**: 同一角色在不同段落音色差异大

**预防策略**:
1. 固定 TTS 模型和参数
2. 记录生成参数供复现
3. CosyVoice 使用相同参考音频

---

### 3.3 长文本截断

**问题**: TTS 单次输入长度限制

**预防策略**:
1. 分句处理（按标点）
2. 句子边界保护
3. 音频片段记录时间戳

---

## 4. 音频处理陷阱

### 4.1 拼接断层

**问题**: 音频拼接处有明显停顿或重叠

**预防策略**:
1. 淡入淡出处理
2. 精确时间戳管理
3. 旁白和对话留白控制

```python
# 音频拼接
segment1 = AudioSegment(...)
segment2 = AudioSegment(...)

# 添加淡入淡出
output = concatenate_with_crossfade(
    segment1.audio,
    segment2.audio,
    crossfade_duration=0.1  # 100ms
)
```

---

### 4.2 格式兼容性问题

**问题**: 不同格式音频参数不一致

**预防策略**:
1. 统一转换为中间格式（WAV）
2. 统一采样率（16kHz/24kHz）
3. 最后转目标格式

---

## 5. 性能陷阱

### 5.1 内存溢出

**问题**: LLM + TTS 同时加载内存不足

**预防策略**:
1. 顺序加载，按需释放
2. 使用量化模型
3. 内存监控和自动清理

```python
class MemoryManager:
    def load_llm(self):
        if psutil.Process().memory_info().rss > 2 * GB:
            self.unload_tts()
    
    def load_tts(self):
        if psutil.Process().memory_info().rss > 2 * GB:
            self.unload_llm()
```

---

### 5.2 处理时间过长

**问题**: 整本小说处理时间不可接受

**预防策略**:
1. 增量处理，支持断点续传
2. 进度显示
3. 并行章节处理（多进程）

---
