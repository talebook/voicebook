# book2audio 功能研究

## 1. 功能分类

### 1.1 核心功能（Table Stakes）

| 功能 | 说明 | 重要性 |
|------|------|--------|
| TXT解析 | 读取和分割小说文本 | 必须 |
| 章节识别 | 自动识别章节标题和边界 | 必须 |
| 角色识别 | LLM识别文本中的角色 | 必须 |
| 对话提取 | 区分对话、旁白、描述 | 必须 |
| TTS合成 | 文本转语音 | 必须 |
| 音频输出 | 生成可播放音频文件 | 必须 |

### 1.2 差异化功能（Differentiators）

| 功能 | 说明 | 价值 |
|------|------|------|
| 年龄-音色映射 | 角色成长时自动切换音色 | ⭐⭐⭐⭐⭐ |
| 音色克隆 | 使用参考音频克隆特定音色 | ⭐⭐⭐⭐⭐ |
| 情感TTS | 根据情绪调整语调语速 | ⭐⭐⭐⭐ |
| 背景音乐 | 自动添加场景氛围音乐 | ⭐⭐⭐ |
| 音效添加 | 笑声、脚步声等 | ⭐⭐⭐ |

### 1.3 不做功能（Anti-Features）

| 功能 | 排除原因 |
|------|---------|
| 视频生成 | 超出范围 |
| 多语言 | 聚焦中文 |
| 实时语音流 | WebSocket复杂 |
| 版权检测 | 用户自行负责 |

## 4. 用户期望

### 4.1 主要用户画像

1. **有声书制作团队**
   - 需求：批量处理、高质量输出
   - 痛点：配音成本高、周期长

2. **个人创作者**
   - 需求：简单易用、快速原型
   - 痛点：技术门槛高

3. **有声读物爱好者**
   - 需求：消费自己喜爱的小说
   - 痛点：现有TTS缺乏情感

### 4.2 核心价值

> 将小说文字转化为有情感、角色分明的声音作品
