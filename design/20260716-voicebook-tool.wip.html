voicebook-tool 首版命令行工具方案

把 EPUB/TXT 书籍转换为多角色、分章节 MP3：先识别角色，再按画像自动选声，支持可编辑脚本、片段级断点续跑与可复现生成。

状态：WIP · 待审查 创建日期：2026-07-16 需求来源：用户 10xdev 请求与 grill 决策 分支：feat/20260716-voicebook-tool
审查门：本文件当前只描述方案。用户显式确认前，不修改业务实现；所有测试结果均为待验证。

1. 原始诉求与验收结果

提供一个可安装的 voicebook-tool 命令行工具，将 EPUB/TXT 书籍转换为纯音频 MP3。工具默认使用 qwen3ttsai.com 的 qwen3tts，也支持 edgetts；默认执行角色识别，并让不同角色自动匹配最合适的音色；输出按章节拆分。

输入
正式支持 EPUB、TXT；默认整本，可按 1,3,8-12 选择章节。
过程
规则/CSI 说话人识别 → 中文角色画像 → 确定性自动选声 → TTS → 统一节奏拼接。
输出
每章一个 MP3、manifest.json、可复用 book.script；--combine 可生成整书 MP3。
可靠性
片段级缓存、失败重试、原子落盘、默认续跑；--force 强制重建。

已确认决策

主题决策原因
默认引擎qwen3tts，映射 qwen3ttsai.com；备选 edgetts沿用已试听的系统音色接入。
服务失败不静默切换引擎避免同一本书前后混入两套角色音色。
输出格式分章 MP3；可选整书 MP3纯音频、易分发、适合断点生产。
CSI 模型不自动下载；显式 models download csi避免转换时突然下载约 650MB 权重。
识别失败[?] 并使用旁白音色安全降级，避免把对白错误绑定给主角。
脚本中文、引擎无关、YAML Front Matter、可人工编辑同一份脚本可生成 Qwen/Edge 两版。
角色画像增加定位、类型、地域；非人类进入方言音色池让机器人、怪兽、地域人物与普通人类形成稳定听觉区分。
主角音可分别保留一个男主角音和女主角音；没有主角时保持未使用保留核心音色，不能被配角抢占。
全书一致性连续发言角色不得同音；同一角色跨章节保持同音，明确年龄变化除外避免对话撞声和章节间角色漂移。
旧入口不提供兼容保证；本次不负责清理新命令按产品契约设计，旧入口后续单独清理。
字幕不纳入首版聚焦可靠的有声书生产主链路。

2. 用户命令契约

# 只识别，不调用 TTS
voicebook-tool inspect book.epub -o book.script

# 从人工校正后的脚本生成分章 MP3
voicebook-tool generate book.script -o output/ --engine qwen3tts

# 二合一：inspect + generate，并保存 output/book.script
voicebook-tool convert book.epub -o output/

# 生成 EdgeTTS 版本与整书 MP3
voicebook-tool generate book.script -o edge/ --engine edgetts --combine

# 选择离散章节；默认整本
voicebook-tool convert book.txt -o output/ --chapters 1,3,8-12

# 忽略缓存，强制重建所选章节
voicebook-tool generate book.script -o output/ --force

# 显式下载 CSI 模型
voicebook-tool models download csi
子命令输入主要输出是否调用 TTS
inspectEPUB/TXTbook.script；EPUB 封面存在时生成同名前缀封面否
generatevoicebook-script v1分章 MP3、manifest、缓存；可选整书 MP3是
convertEPUB/TXT中间脚本 + generate 的全部输出是
models download csi模型名models/csi-v1/否

退出行为

  • 全部所选章节成功才返回 0。
  • 单片段短暂失败按引擎策略重试；重试耗尽后章节记为失败，继续其他独立章节，最终非 0。
  • 认证、欠费、协议不兼容等系统性错误立即停止，不自动切换到 Edge。
  • 所有失败写入 manifest；重跑同一命令时只补缺失或失效的片段。

5. EPUB/TXT 解析

EPUB

  1. 使用标准库读取 ZIP，解析 META-INF/container.xml 与 OPF；不把归档直接解压到磁盘。
  2. 按 OPF spine 阅读顺序遍历内容项，忽略 nav、封面页和 linear="no" 项。
  3. 从 Dublin Core 元数据读取书名、作者、语言；从 manifest 的 cover 属性或旧 cover meta 定位封面。
  4. 优先按正文中的章节标题拆分；无可靠标题时，一个有效 spine 文档作为一章，并使用 TOC/标题/文件名兜底。
  5. 保留段落顺序、中文引号和必要标点，剥离脚本、样式、隐藏节点和纯导航文本。

TXT

  • 兼容现有 第X章 中文标题。
  • 没有任何章节标题时，整份文件作为第 1 章，标题使用文件名。
  • 尝试 UTF-8/UTF-8 BOM；其他编码给出明确错误，不静默乱码。
安全边界：限制 EPUB 归档总展开体积、单项体积和内容项数量；规范化 ZIP/OPF 相对路径；输出文件名删除路径分隔符和控制字符，并限制长度。

6. 角色识别、自动选声与语速

阶段行为降级方式
名字与对白复用规则 R1–R8；检测到本地 CSI 时做融合兜底CSI 不存在时继续规则识别，并在脚本/manifest 标记降级。
角色画像全书范围汇总定位、类型、性别、年龄、长期地域和音色描述证据未知值保持未知；不把当前旅行地点误写成角色长期地域。
主角判断按章节覆盖、出现次数和对白占比保守判断，最多一个男主角和一个女主角没有显著领先者则不设置主角；脚本人工定位优先。
音色匹配先做语言/类型/性别硬过滤，再做年龄、地域、描述评分，最后执行全书冲突约束扩大同声别候选范围，不跨性别、不占用主角保留音。
人工覆盖脚本中的引擎覆盖优先于自动匹配音色不存在时生成前失败，列出可用候选。
语速自动 使用画像/引擎默认；xN 显式覆盖范围外直接报脚本行号错误，不擅自截断用户值。

项目分值/约束
人工覆盖硬指定，不参与评分；仍校验音色存在。
语言、角色类型、明确性别硬过滤；非人类进入方言池；明确性别不跨声别。
性别匹配+40；中性/未知不加分。

  1. 先锁定人工覆盖和按引擎配置的男/女主角保留音；没有对应主角时保留音保持未使用。
  2. 按章节覆盖率、对白字数、角色定位和发言邻接度排序：主角 → 重要角色 → 高邻接角色 → 配角。
  3. 扫描全书发言顺序建立角色邻接图；连续出现的两个不同角色形成“不得同音”边。
  4. 使用确定性候选排序与有限回溯分配：优先最高分，同时满足邻接不同音和主角音保留。
  5. 同一规范角色 ID 只分配一次，跨章节复用；别名先归一。角色@老年 等明确年龄变化创建变体并允许换音，情绪变化不换音。
  6. 候选冲突无法满足时扩大同声别年龄/地域范围并记录警告；不得偷偷使用主角保留音或跨性别。

每个角色的候选音色、得分、选中原因、地域证据、邻接冲突和最终音色都写入 manifest，便于解释与复现。Edge 使用同一约束算法，但候选来自扩充后的中文 Neural voice 目录。

统一节奏基线：标题后 900ms、逻辑片段间 250ms、章尾 700ms；API 长文本切块之间不额外插入片段停顿。Qwen WAV 继续做边缘平滑；Edge 片段在最终装配前规范化为统一 PCM，再编码为章节 MP3。

指纹与恢复

  • 片段指纹包含：脚本版本、规范化文本、角色/状态、引擎、音色、最终语速和音频处理版本。
  • 章节指纹包含有序片段指纹、停顿配置和编码配置。
  • 文件存在且指纹一致才可复用；仅文件名相同不能视为命中。
  • manifest 使用临时文件 + 原子替换；章节 MP3 也先写临时文件，成功后替换正式产物。
  • --force 忽略所选章节的缓存，但不删除未选择章节的已完成结果。

完整片段正文保留在 book.script；manifest 只记录角色、文本哈希、短预览、时长和状态，避免大型小说重复保存全部正文。

8. MP3 与元数据

  • 章节音频统一编码为单声道 MP3；具体采样率和码率在实现时固定并写入 manifest。
  • ID3：title=章节标题、album=书名、track=章节序号/总数、artist=作者。
  • EPUB 有封面时通过 ffmpeg 写入 attached picture；它是 MP3 封面元数据，不是视频流。
  • --combine 使用相同编码参数生成整书 MP3，并写入整书级 ID3；分章文件仍保留。
  • 生成完成后用 ffprobe 校验 codec、时长和元数据，不能只检查文件非空。

10. 性能、安全与外部依赖

大书规模
按既有 research 的 2,502 个 EPUB 内容项、约 663 万汉字规模设计；不把全书音频放入内存，manifest 增量更新。
并发
章节顺序处理，片段内部有限并发；Qwen 与 Edge 分别使用保守并发上限，避免触发限流。
网络与隐私
两个引擎均为云端 TTS，正文会发送到外部服务；README 和首次执行提示必须明确说明。
工具依赖
环境由 uv 管理；ffmpeg/ffprobe 是显式系统依赖，启动前检测并给出安装提示。
YAML 安全
只使用 safe loader，并对最终数据类型、键名、枚举和长度再次校验。
第三方服务
qwen3ttsai.com 当前曾出现上游欠费；错误必须透明呈现，不能把端点存在写成服务 SLA。

必须覆盖的自动化场景

层级场景通过标准
单元章节表达式1,3,8-12 正确、去重、有序；越界和非法值报错。
单元EPUB/TXT 解析按 spine 顺序提取两章、元数据和封面；无标题 TXT 产生单章。
单元脚本 round-trip中文元数据、主角音、定位、类型、地域、x0.9、音色覆盖和标签写入后可等价读回。
单元主角保留音主角使用配置音；保留音不分给配角；没有对应主角时保留音保持未使用。
单元连续发言与跨章一致性邻接角色音色不同；同一规范角色跨章节同音；年龄变体允许换音。
单元类型/地域选声非人类只进方言池；长期地域触发对应候选；旅行地点不改音；明确性别不跨声别。
单元海外音色门禁只有通过中文可懂度白名单的海外音色可自动使用；国家无对应性别时优先保持性别。
单元自动选声确定性覆盖值优先；相同脚本/目录版本的候选得分、冲突处理和最终结果完全一致。
单元错误分类429/暂时性 5xx 重试；欠费/认证错误不重试整本且不切引擎。
单元缓存恢复第二次同配置零 TTS 调用；只改一段只重生一段；--force 重建所选章。
集成离线假 TTS + ffmpeg生成两章可解码 MP3；留白、语速、ID3、封面和时长均可由 ffprobe 验证。
CLIinspect/generate/convert输出结构和退出码符合契约；convert 保留脚本。

外部冒烟测试

  • EdgeTTS：用极短公开测试文本生成一章；网络不可用时记录未执行原因。
  • Qwen3TTS：只在 qwen3ttsai.com 恢复可用时执行极短冒烟；若仍为 Arrearage，记录为外部服务限制，不伪造成功。
  • 真实 EPUB：只使用仓库内可合法测试的样本，检查章节顺序、标题和封面。
