VOICEBOOK · PACING DESIGN

Qwen 多角色音频节奏修正

在保留片段边界去爆音的基础上，为逻辑片段恢复自然留白，并校正老年角色偏慢的听感。

ACTIVE2026-07-16影响：Qwen 合成、角色选声、Demo MP4
需求复述：当前完整版 MP4 虽已消除边界“啪”声，但旁白与对白、不同角色之间无停顿，节奏过紧；老年角色又明显过慢。需要在不吞字、不重新引入爆音的前提下生成更自然的可播放版本，并说明旧版本是否人为改慢。

01 · 已确认事实与工作假设

类型内容依据
事实Qwen 片段当前由 concat demuxer 直接拼接，没有插入静音。synth_chapter_qwen 只列出生成的 WAV。
事实Qwen 请求没有语速字段，仅发送 text、voice、mode。QwenTTSClient.generate 的请求体。
事实旧 Qwen MP4 没有额外降速；慢速来自自动选择的老年系统音色。Qwen voice spec 只有音色名，未执行 tempo 滤镜。
旁支事实Edge 老年音色配置为 -10%，但与本次 Qwen MP4 无关。VOICE_BUCKETS。
已实现参数250ms 角色留白、450ms 标题留白、700ms 章节结尾留白和 1.15× 老年语速用于本轮成品。自动测试与媒体检查通过；主观听感仍接受用户反馈。

02 · 决策

按逻辑片段留白。 标题后插入 450ms 静音；旁白/对白或角色片段之间插入 250ms 静音；每章结尾插入 700ms 静音，隔开下一章标题。因 API 1000 字限制产生的同一逻辑片段内部切块不插入停顿。
老年画像明确加速。 Qwen API 不支持 rate，因此在本地对老年角色 WAV 使用 1.15× tempo 后处理；其他年龄保持 1.00×。处理后再执行边缘淡入淡出。
保留旧资源。 新成品与节奏评测材料进入 design/resources/qwen-segment-pacing/，不覆盖此前用于去爆音对照的 MP4。
开放反馈：250ms / 450ms / 1.15× 是当前工作值，不是 API 固有限制。用户给出更松或更快的偏好时先回写本方案，再改实现。

03 · 目标流程

逻辑片段（标题 / 旁白 / 角色对白）
  ├─ 按 API 上限切块并并发合成
  ├─ 老年角色：tempo 1.15×；其余：1.00×
  ├─ 每个语音块：10ms 淡入 + 5ms 淡出
  ├─ 同一逻辑片段内部：连续拼接
  ├─ 逻辑片段边界：标题后 450ms / 其他 250ms 静音
  └─ 章节结尾：700ms 静音
                         ↓
                 章节 WAV → AAC / MP4

04 · 实现边界与兼容性

  • Qwen voice spec 增加本地 tempo 值；系统音色覆盖仍可用，年龄画像继续决定默认 tempo。
  • 静音 WAV 复用相邻 Qwen WAV 的采样率、声道和位深，避免 concat 格式不一致。
  • Edge 与历史 CosyVoice 路径不改变；Qwen API 协议不改变。
  • 章节时长和章节元数据会随新增留白重新计算。
  • 回滚只需恢复 tempo 为 1.00、pause 为 0；旧 MP4 资源继续保留。
